# 카드풀 선택 UI 구현 메모

## 구현된 직접 선택
- 카드를 소환하거나 주문을 사용할 때 비용으로 소진할 준비 마나를 플레이어가 직접 선택합니다.
- 공명은 실제로 선택해 소진한 마나의 카드군만 참조합니다.
- 썰물, 역류, 사막화는 전장의 유효한 몬스터를 직접 대상으로 지정합니다.
- 성스러운 거울의 벽은 상대 라이프의 뒷면 카드 위치를 직접 선택합니다.
- 직접 공격도 파괴할 상대 라이프를 직접 선택합니다. 추가 라이프 손실이 있으면 필요한 장수를 모두 고릅니다.
- 파묘는 묘지로 보낼 준비 마나와 손으로 되돌릴 묘지 카드를 각각 선택합니다. 방금 묘지로 보낸 마나도 되가져올 수 있습니다.
- 신전의 유망주는 가져올 라이프를 선택한 뒤, 새로 공개된 카드를 포함한 손에서 라이프로 되돌릴 카드를 선택하거나 건너뜁니다.
- 물결을 읽는 자는 확인한 덱 위 카드를 유지할지 묘지로 보낼지 선택합니다.
- 몰아치는 파도는 확인한 카드가 소환 가능할 때 소환 여부를 선택합니다.
- 불타는 행렬은 공개된 카드 중 실제로 소환할 카드를 빈 슬롯 수 이내에서 선택합니다.
- 각성으로 발동한 성스러운 거울의 벽도 효과 조종자가 상대 라이프를 선택합니다.

## 서버 판정
- 마나 수, 중복 선택, 준비/소진 상태, 대상의 소유자와 영역, 대상 조건을 서버에서 다시 검증합니다.
- 선택이 필요한 효과가 남아 있는 동안 다른 게임 행동은 처리되지 않습니다.
- 선택 중에는 턴 제한 시간을 멈추고, 모든 선택이 끝난 뒤 현재 턴의 시간을 새로 시작합니다.
- 덱 위 공개 카드는 선택하는 플레이어에게만 보이며 상대에게는 카드 정체가 공개되지 않습니다.

## 대전 화면 UI 개편
- 데스크톱에서는 상단 바를 포함한 전체 대전판이 `100svh` 안에 들어오며 문서 전체의 세로 스크롤을 사용하지 않습니다.
- 기존 좌우 방 정보·행동 패널을 제거하고, 방 관련 기능은 상단 `⋮` 메뉴로 옮겼습니다.
- 라이프는 숫자로 대체하지 않고 양쪽 전장 왼쪽의 실제 뒷면 카드 열로 표시합니다. 소유자도 정체를 볼 수 없습니다.
- 마나는 전장 오른쪽의 축약 토큰으로 표시하며 카드명, 비용, 카드군, 준비/소진 상태를 확인할 수 있습니다.
- 공개 카드와 마나에 마우스를 올리거나 키보드 포커스를 두면 전체 카드 상세가 표시됩니다. 카드를 클릭하면 상세창이 고정되고 `Esc`로 닫힙니다.
- 양쪽 묘지는 덱 옆의 공개 카드 더미로 항상 표시됩니다. 묘지 더미나 상단 요약의 묘지 버튼을 누르면 어느 턴이든 전체 묘지를 열람할 수 있습니다.
- 묘지 창은 최근에 들어간 카드부터 표시하며, 파묘의 카드 선택에도 같은 창을 사용합니다.
- 공격자, 공격 대상, 주문 대상, 비용 마나, 라이프 선택은 선택 가능·선택됨·소진 상태를 서로 다른 테두리와 밝기로 표시합니다.
- 내 손패만 하단 가로 영역에 크게 표시하며, 카드가 많아지면 손패 영역만 가로 스크롤합니다.


## 배포 준비

- 실시간 서버 주소는 `VITE_GAME_SERVER_HOST`로 주입됩니다.
- 로컬 접속(`localhost`, `127.0.0.1`)에서는 값이 없을 때 `localhost:8787`을 사용합니다.
- 서버는 PartyServer 기반 Cloudflare Worker와 SQLite Durable Object로 직접 배포됩니다.
- 방은 WebSocket 하이버네이션을 사용하며, 유휴 상태에서도 저장된 게임 상태를 복원할 수 있습니다.
- GitHub Pages의 하위 경로는 `VITE_BASE_PATH`로 처리합니다.
- 자세한 최초 배포 순서는 `DEPLOY-FRIENDS.md`를 참고하세요.

## 확장형 콘텐츠·포맷 구조

- 표시 이름과 패키지 이름을 `Duel Spirits` / `duel-spirits`로 변경했습니다. 기존에 배포한 Worker 주소와 로컬 저장 키는 호환성을 위해 유지합니다.
- 카드 정의를 `src/content/cards.ts`로 옮기고 카드 인스턴스와 분리했습니다.
- 각 카드에 세트 ID, 수집 번호, 콘텐츠 버전을 추가했습니다.
- 모든 카드 해금 정책을 `AllCardsUnlockedPolicy`로 분리했습니다. LCG 세트 권한, 캠페인 해금, CCG 보유 수량 정책 구현체도 준비했습니다.
- 저장 덱 스키마를 v2로 올리고 포맷, 선택 세트, 드래프트 풀을 저장합니다. 기존 저장 덱은 자동 이전됩니다.
- 전체 카드전, 세트 한정전, 드래프트, 금지·제한전, 캠페인 전용 포맷을 추가했습니다.
- 덱 빌더에서 포맷 선택, 세트 선택, 드래프트 풀 생성과 포맷별 카드 수량 검증을 지원합니다.
- 방 생성 시 PvP 포맷을 선택하며, 서버는 방 포맷과 제출 덱 포맷의 일치를 검증합니다.
- 경기 상태에 콘텐츠·규칙 버전, 난수 시드, 행동 순번을 저장합니다.
- Cloudflare Durable Object는 성공한 행동을 순번과 시각과 함께 기록합니다.
- 캠페인 시나리오, 사람·AI 조종자 경계, LCG/CCG 인벤토리 타입을 추가했습니다.
- 상세 설명은 `ARCHITECTURE.md`에 있습니다.
